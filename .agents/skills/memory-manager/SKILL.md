# 记忆管理器

本 Skill 用于把“你已经提醒过很多次”的内容沉淀为可复用记忆。

## 使用方式

1. 使用下方模板生成或更新 `memory.md`
2. 只记录稳定事实、长期约束、明确决策、已证实的坑点
3. 为每条记忆写上来源和更新时间
4. 临时讨论、易失状态、短期计划不要写进 memory
5. 若某条记忆已经失效，必须显式修订或删除

## 适合写入 memory 的内容

- 用户长期偏好
- 项目稳定约束
- 关键设计决策
- 经常漏掉的验收点
- 事故复盘结论

## 不适合写入 memory 的内容

- 今天临时要做的任务
- 尚未确认的猜测
- 随时会变化的运行状态

---

## 模板

```markdown
# 项目记忆

## 1. 稳定事实

- [事实]（来源 / 更新时间）

## 2. 设计决策

- [决策]（原因 / 影响 / 更新时间）

## 3. 常见坑与防线

- [坑点]（如何避免）

## 4. 用户偏好

- [偏好]（何时适用）

## 5. 需要持续验证的风险

- [风险]（是否已转为 eval 或验收项）
```

## 使用说明

本模板是项目内参考模板，不要求所有模块都机械填满所有章节。  
`spec-validator` 会按“通用必填项 + 模块类型专项项 + 交付等级附加项”检查完整性。

- `页面类`：重点写界面结构、交互流程、状态处理、导航和视觉约束
- `API 类`：重点写接口契约、错误处理、权限、数据影响和验证方式
- `数据处理类`：重点写输入源、触发方式、容错 / 重试、输出和结果校验
- `工具类`：重点写命令入口、参数 / 配置、输出格式、安装 / 运行体验

# 记忆管理器

本 Skill 用于把"你已经提醒过很多次"的内容沉淀为结构化、可检索、可追溯的项目记忆。

## 使用方式

1. 使用下方模板生成或更新 `.ai-os/memory.md`
2. 每条记忆必须有**唯一 ID**、**分类**、**来源**、**状态**和**更新时间**
3. 只记录稳定事实、长期约束、明确决策、已证实的坑点
4. 临时讨论、易失状态、短期计划不要写进 memory
5. 若某条记忆已经失效，将状态标记为 `已过时`，附注原因，不要直接删除

## 适合写入 memory 的内容

- 架构决策（选型理由、方案对比、影响范围）
- 编码约定（命名规范、文件组织、API 风格）
- 已知坑点（踩过的坑、绕行方案、触发条件）
- 用户偏好（交互偏好、风格偏好、流程偏好）
- 硬性约束（技术限制、外部依赖限制、合规要求）

## 不适合写入 memory 的内容

- 今天临时要做的任务
- 尚未确认的猜测
- 随时会变化的运行状态
- 已记录在 `.ai-os/STATE.md` 中的当前进度

## 记忆条目规范

每条记忆必须包含：

| 字段 | 必填 | 说明 |
|------|------|------|
| ID | 是 | 唯一标识，格式为 `{分类缩写}-NNN`（如 `AD-001`、`CV-003`） |
| 内容 | 是 | 记忆的具体内容 |
| 分类 | 是 | architecture / convention / pitfall / preference / constraint |
| 来源 | 是 | 产生这条记忆的事件（事故复盘 / 用户提醒 / 开发发现） |
| 状态 | 是 | 生效中 / 待确认 / 已过时 |
| 日期 | 是 | 创建或最后更新日期 |

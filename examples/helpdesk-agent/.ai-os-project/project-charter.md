# Helpdesk Agent 项目章程

## 1. 项目概述

- **项目目标**：实现一个帮助客服团队做首轮工单分诊的 Agent
- **目标用户**：客服主管、值班客服
- **项目类型 / archetype**：AI Agent / Workflow 应用
- **交付等级**：L2 标准
- **成功指标**：80% 常见工单可被正确分类；高风险工单全部进入人工复核

## 2. 范围边界

### 范围内
- 工单文本分类
- 优先级建议
- 高风险关键词人工转交
- 分诊结果审计记录

### 范围外
- 自动回复用户
- 外部知识库检索
- 多语言支持

## 3. 关键场景

- **场景 A**：客服收到新工单后，系统自动给出分类和优先级
- **场景 B**：工单出现退款、账号封禁等敏感词时，系统阻止自动流转并要求人工复核

## 4. 非功能需求

- **性能**：单次分诊响应时间 < 3 秒
- **安全**：敏感关键词命中时必须走人工确认
- **稳定性 / SLO**：工作时间成功率 >= 99%
- **成本 / 资源预算**：单工单平均推理成本可监控
- **兼容性**：支持 Web 后台接入

## 5. 里程碑

| 里程碑 | 目标 | 交付物 | 截止条件 |
|--------|------|--------|----------|
| M1 | 完成分诊核心链路 | spec、tasks、分诊服务、验收项 | 本地验证通过 |
| M2 | 完成交付与回归 | release-plan、evals、review-report | 可进入人工试用 |

| 模块 | 目标 | 依赖 | 交付等级 | 后续 spec |

## 7. 外部依赖

- 客服系统的工单 webhook
- LLM 推理服务
- 命中人工复核时的通知通道

## 8. 审批点

- 高风险关键词策略上线
- 生产环境工单写回
- 费用型外部模型调用

# ticket-triage 需求规格

- **关联项目章程**：`project-charter.md`
- **所属里程碑**：M1
- **范围外内容**：自动回复、多语言支持

## 概述

- **模块用途**：对新工单进行分类、优先级判断，并在高风险场景转人工
- **目标用户**：客服主管、值班客服
- **核心价值**：降低首轮人工分诊成本，同时避免敏感工单被错误自动流转

## 页面/接口清单

| 编号 | 页面/接口 | 类型 | 描述 |
|------|----------|------|------|
| A-001 | 工单分诊接口 | API | 输入工单文本，输出分类与优先级 |
| P-001 | 分诊结果查看页 | 页面 | 展示分类、优先级和人工复核标记 |

### 页面跳转关系
```
[工单列表] → [分诊结果详情]
```

## 功能需求

### FR-001: 工单分类

**描述**：系统根据工单文本给出 `billing`、`technical`、`account` 等分类。

**交互流程**：
1. 用户提交工单文本
2. 系统执行分类
3. 系统返回分类结果与置信说明

**数据字段**：

| 字段名 | 类型 | 必填 | 校验规则 | 展示格式 | 默认态/占位符 | 说明 |
|--------|------|------|---------|---------|-------------|------|
| ticket_text | string | 是 | 10-2000 字符 | 原文显示 | “请输入工单内容” | 工单正文 |
| category | string | 是 | 枚举值 | 标签 | - | 分诊分类 |
| priority | string | 是 | low/medium/high | 标签 | - | 优先级 |
| requires_human_review | boolean | 是 | true/false | 标签 | - | 是否人工复核 |

**状态处理**：
- 加载中：显示“正在分诊”
- 空数据：提示输入工单内容
- 错误：提示“分诊失败，请稍后重试”
- 成功：展示分类、优先级、人工复核标记

**权限要求**：客服与客服主管可查看，只有主管可修改规则

**边界条件**：
- 工单文本为空：拒绝提交
- 命中高风险关键词：强制 `requires_human_review = true`

## 数据模型

### triage_result 表

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | 自增主键 | PK | - | 主键 |
| ticket_id | 字符串(64) | NOT NULL | - | 工单 ID |
| category | 字符串(32) | NOT NULL | - | 分类 |
| priority | 字符串(16) | NOT NULL | medium | 优先级 |
| requires_human_review | 布尔 | NOT NULL | false | 是否人工复核 |
| created_at | 时间戳 | NOT NULL | 当前时间 | 创建时间 |

### 索引

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_ticket_id | ticket_id | 唯一 | 工单唯一 |
| idx_requires_human_review | requires_human_review | 普通 | 人工复核筛选 |

### 关联关系
- 工单系统 1 : 1 triage_result（通过 `ticket_id` 关联）

## API 定义

### A-001: 工单分诊接口

- **URL**: `[POST] /api/v1/tickets/triage`
- **权限**: 需要登录

**请求参数**：

| 参数名 | 位置 | 类型 | 必填 | 说明 |
|--------|------|------|------|------|
| ticket_id | body | string | 是 | 工单 ID |
| ticket_text | body | string | 是 | 工单正文 |

**成功响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "category": "billing",
    "priority": "high",
    "requires_human_review": true
  }
}
```

**错误响应**：

| 错误码 | HTTP状态 | 说明 |
|--------|---------|------|
| 40001 | 400 | ticket_text 不合法 |
| 42201 | 422 | 输出结构校验失败 |

## 非功能需求

- **性能**：单次分诊 < 3 秒
- **安全**：敏感工单必须进入人工复核
- **兼容**：支持现有客服后台接入

## 关联模块

| 模块 | 关系 | 说明 |
|------|------|------|
| ticket-webhook | 依赖 | 提供工单输入 |
| review-queue | 被依赖 | 接收人工复核任务 |

## 验收标准

### 正常流程
- [ ] **AC-001**：Given 普通咨询工单, When 调用分诊接口, Then 返回合法分类和优先级
- [ ] **AC-002**：Given 含敏感关键词工单, When 调用分诊接口, Then `requires_human_review = true`

### 异常流程
- [ ] **AC-E001**：Given 空文本, When 调用接口, Then 返回参数错误
- [ ] **AC-E002**：Given 模型输出缺字段, When 结构化校验失败, Then 返回错误并记录日志

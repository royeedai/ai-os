# AI-OS Examples

这里放 AI-OS 母仓库层面的使用示例。

这些示例的目的不是提供业务代码模板，而是说明：

- 哪类问题更适合通过“用法和项目规划”解决，而不是继续加 framework 规则
- 遇到不同结构特征的项目时，AI-OS 应该如何选择更合适的 workflow、工件和交付深度

## 当前示例

- `platform-project-foundation-first.md`
  - 说明平台型 / 后台型 / 流程型项目为什么要先做基础能力层和首条闭环
- `platform-project-foundation-first/`
  - 对应的示例项目工件骨架
- `minimum-sufficient-change.md`
  - 说明小范围改动为什么不该默认升级成完整重流程
- `minimum-sufficient-change/`
  - 对应的示例项目工件骨架

## 使用方式

当你在判断“这个需求该怎么用 AI-OS 承接”时，先看：

1. 它更像哪一个示例
2. 示例里推荐的 workflow 是什么
3. 示例里要求的工件和证据深度是什么
4. 如果当前问题和示例差别很大，再考虑是否需要调整 framework

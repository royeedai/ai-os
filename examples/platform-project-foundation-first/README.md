# Example Skeleton: Platform Project Foundation First

这个目录展示一个“平台型 / 后台型 / 流程型项目”的 AI-OS 工件骨架。

它要表达的重点不是业务细节，而是：

- 第一阶段目标不是“先做一个业务模块页面”
- 而是先把共享基础能力和首条可运行闭环规划清楚

建议结合上层说明一起看：

- [../platform-project-foundation-first.md](../platform-project-foundation-first.md)

## 示例包含

- `.ai-os/project-charter.md`
- `.ai-os/tasks.yaml`
- `.ai-os/acceptance.yaml`
- `.ai-os/STATE.md`

## 这个示例想说明什么

1. `project-charter` 里会先声明共享基础能力依赖和第一阶段里程碑
2. `tasks.yaml` 会把基础能力层排在业务模块前面
3. `acceptance.yaml` 会把第一阶段验收写成“闭环可运行”，而不是“页面已完成”

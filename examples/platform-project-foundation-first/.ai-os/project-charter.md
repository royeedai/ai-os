# 项目章程（示例）

## 1. 项目概述

- **项目名称**：Warehouse Hub
- **项目类型**：平台型后台系统
- **目标市场 / 主要使用地区**：中国大陆仓储与运营团队
- **体验 / 视觉参考方向**：信息密度较高的国内业务后台
- **默认交付等级**：L2 标准
- **共享基础能力依赖结构**：是

## 2. 当前阶段目标

当前阶段不是“完成某个业务模块页面”，而是：

- 登录、权限、导航骨架可用
- 组织 / 仓库上下文可切换
- 多语言框架可工作
- 基础设置与数据字典可支撑首条业务闭环
- 首条核心业务链路可从入口走到结果

## 3. 里程碑

| 里程碑 | 目标 | 完成标准 |
| --- | --- | --- |
| M0 | 基础能力层 | 登录、权限、导航骨架、组织 / 仓库上下文、多语言、基础设置可用 |
| M1 | 首条业务闭环 | 以“入库登记”为例，能从列表进入、完成提交、看到结果 |
| M2 | 业务模块扩展 | 在基础能力已稳定的前提下扩展库存、出库、调拨等模块 |

## 4. 核心模块

| 模块 | 模块类型 | 说明 |
| --- | --- | --- |
| 基础能力层 | 页面类 / API 类 | 登录、权限、导航、多语言、组织 / 仓库上下文 |
| 基础设置 | 页面类 / API 类 | 字典、仓库配置、状态枚举 |
| 入库登记 | 页面类 / API 类 | 首条核心业务闭环 |

## 5. 不在当前阶段做

- 不按业务模块列表顺序机械推进
- 不在基础能力缺失时直接判定业务模块“已完成”
